# Гостемания — Документация проекта

## Стек и структура файлов

```
/
├── index.html              # Всё приложение — один HTML-файл, 10 страниц внутри
├── css/style.css           # Все стили (шрифты, компоненты, анимации)
├── js/app.js               # Роутер + все интерактивы
├── Fonts/
│   ├── FLAME-REGULAR.OTF   # Шрифт Flame для заголовков
│   └── FLAME-BOLD.OTF      # Жирный Flame
├── pictures/
│   └── hero_1.png          # Фоновое фото на главном экране
├── images/                 # Статичные изображения (добавляются вручную)
├── videos/                 # Видеофайлы (добавляются вручную)
└── audio/                  # Аудиофайлы (добавляются вручную)
```

**Tailwind CDN** — подключён через `<script src="https://cdn.tailwindcss.com">`, кастомные цвета прописаны в `tailwind.config` прямо в HTML.  
**Roboto** — подключён через Google Fonts.  
**Vanilla JS** — никаких фреймворков, всё в `js/app.js`.

## Архитектура SPA

Всё приложение — **один HTML-файл** с 10 страницами (`div.page`). Активная страница получает класс `active` (CSS: `display: block`), остальные скрыты (`display: none`).

### Страницы (`id`)

| ID             | Название                   | Порядок в курсе |
|----------------|----------------------------|-----------------|
| `page-home`    | Главная                    | —               |
| `page-block1`  | Философия Гостемании       | 1 / 5           |
| `page-block2`  | Алгоритм ДОБРО             | 2 / 5           |
| `page-block3`  | Работа с инцидентами       | 3 / 5           |
| `page-hub`     | Замена и Комплемент (хаб)  | 4 / 5           |
| `page-section1`| Замена + Комплемент        | подраздел 4     |
| `page-section2`| Комплемент                 | подраздел 4     |
| `page-section3`| Особые случаи              | подраздел 4     |
| `page-block8`  | Заключение                 | 5 / 5           |

### Роутер (`navigateTo`)

```js
navigateTo('block2')
```

1. Снимает класс `active` со всех страниц
2. Добавляет `active` на целевую страницу
3. Скроллит вверх
4. Обновляет навигационную строку (название главы, прогресс X/5, ширину прогресс-бара)
5. Запускает `initFadeIn()` через 50 мс
6. Вызывает специфичные инициализаторы:
   - `block2` → `initGallery()`
   - `block3` → `initSortable()` + `initVideoQuiz()`
   - `hub`    → `initHubWarmup()`

## Интерактивы

### 1. Drag-and-drop: сортировка фраз (Блок 3)

**ID списка:** `sortable-list`  
**Правильный порядок:** `[1, 3, 0, 2]` (0-индекс по `data-idx`)  
Что это: «Вы правы» → «Простите» → «Проверю заказ» → «Комплемент»

- Реализован через HTML5 drag-and-drop + touch-события
- Функция проверки: `checkSortOrder()` — читает порядок `data-idx`, сравнивает с `CORRECT_ORDER`
- Результат: показывает `#sort-feedback` (`.correct` / `.incorrect`)
- После проверки: открывается `#sort-analysis` (разбор шагов) и кнопка «Продолжить» → открывает `#video-section`

### 2. Видеовикторина: определение настроения (Блок 3)

**Шаги:** 3 видео, последовательно  
**Правильные ответы:** `['спокойный', 'расстроенный', 'злой']` (константа `VIDEO_ANSWERS`)  
**Фидбэк-тексты:** константа `VIDEO_FEEDBACK` (3 строки)

- Функция: `answerVideo(btn, stepIdx, answer)`
- Неверный ответ → кнопка мигает красным (`flash-wrong` анимация), можно попробовать снова
- Верный ответ → показывает `#vq-fb-N` с текстом, через 1 сек переходит к следующему шагу
- После 3-го шага: `showVideoStep(3)` → открывает `#video-quiz-done`

### 3. Хаб: 3-шаговая разминка

**Индикатор:** `.warmup-step-dot` × 3 (оранжевый = активный, зелёный = пройден)  
**Шаги:**

| Шаг | ID       | Тип                          | Функция проверки   |
|-----|----------|------------------------------|--------------------|
| 1   | `hub-ws0`| Drag-and-drop по 2 зонам     | `checkHubWarmup0()`|
| 2   | `hub-ws1`| Matching (3 вопроса)         | авто при последнем правильном |
| 3   | `hub-ws2`| Drag-and-drop по 2 зонам     | `checkHubWarmup2()`|

- Прогресс: `hubWarmupStep` (0–3)
- Переход между шагами: `advanceWarmup(fromStep)` — через `setTimeout` 1600мс после верного ответа
- После 3-го шага: появляется `#hub-warmup-done` с поздравлением
- Зоны инициируются в `initHubWarmup()` через `initZoneSort()`

### 4. Matching (шаг 2 разминки)

**Правильные ответы:** `MATCH_ANSWERS = ['десерт', 'напиток-десерт', 'бургер-сет']`  
Соответствует жалобам: грязь → десерт, ожидание → напиток+десерт, грубость → бургер-сет

- Функция: `pickHubMatch(btn, qIdx, answer)` 
- Неверный ответ → мигает красным
- Все 3 решены → показывает фидбэк → `advanceWarmup(1)`
- Состояние: `hubMatchSolved = [false, false, false]` (сбрасывается при `initHubWarmup`)

### 5. Хаб: разблокировка подразделов

**Состояние:** `hubDone = [false, false, false]` (глобальный массив)

1. Изначально `hub-card-2` и `hub-card-3` имеют класс `locked` (CSS: `opacity: 0.45`, `pointer-events: none`)
2. `completeSection(n)` → `hubDone[n-1] = true` → добавляет `.done` карточке N → снимает `.locked` с карточки N+1 → `navigateTo('hub')`
3. Когда `hubDone.every(Boolean)` → показывается `#hub-next-row` (кнопка «Далее» → block8)

### 6. Галерея (Блок 2)

- 5 слайдов, CSS transform `translateX(-N*100%)`
- Состояние: `galleryIdx` (0–4)
- `galleryMove(dir)` — переключение ±1
- `renderGallery()` — обновляет трек и точки

### 7. FAQ-аккордеон

- `toggleFaq(btn)` — переключает классы `open` на `btn` и следующем сиблинге `.faq-body`
- CSS: `.faq-body { max-height: 0 }` / `.faq-body.open { max-height: 800px }` с transition

## Анимация fade-in по скроллу

Все блоки контента имеют класс `fade-in` (opacity: 0, translateY: 28px).

```js
initFadeIn()  // вызывается при каждом navigateTo() через setTimeout 50мс
```

Использует `IntersectionObserver` с `threshold: 0.08`. Элементы уже в viewport получают `.visible` сразу, остальные — при скролле.
