# Прогресс курса и SCORM — как это устроено и что учитывать

Этот файл — памятка на будущее: как в курсе сохраняется прогресс, как это
связано со SCORM, и на что обращать внимание при доработках.

---

## 1. Что и где хранится

Состояние прогресса — это один маленький JSON-объект:

```json
{ "unlocked": 3, "hub": [true, false, false] }
```

- `unlocked` — сколько глав открыто на главной странице (1…5). Главы
  открываются последовательно: посмотрел главу — открылась следующая.
- `hub` — три флага прохождения подразделов блока 4 (Замена / Комплемент /
  Особые случаи).

Этот JSON пишется **в двух местах** (см. `js/app.js`, функции `saveProgress` /
`loadProgress`):

| Хранилище | Когда используется | Поле |
|-----------|--------------------|------|
| **SCORM** `cmi.suspend_data` | когда курс запущен внутри LMS | основной |
| **localStorage** браузера | когда курс открыт вне LMS (GitHub Pages, локально) | запасной |

При загрузке курс сначала пытается прочитать состояние из SCORM, а если его нет —
из localStorage. Поэтому курс одинаково работает и в LMS, и просто в браузере.

---

## 2. Как это связано с упаковкой `scorm_pack.py`

SCORM-обёртка **не лежит в репозитории** — она добавляется автоматически при
запуске `python scorm_pack.py`. Скрипт:

1. кладёт рядом файл `scorm_api.js` (обёртка над SCORM 1.2 API LMS);
2. вставляет `<script src="scorm_api.js"></script>` в `<head>` index.html;
3. собирает всё в ZIP с `imsmanifest.xml`.

Внутри `scorm_api.js` создаётся глобальный объект `window.SCORM` с методами
`init / get / set / commit / finish / complete`. Именно его использует `app.js`.

> Важно: пока курс открыт **не из ZIP** (например, на GitHub Pages),
> `window.SCORM` не существует — поэтому в `app.js` все вызовы SCORM обёрнуты в
> проверку `if (window.SCORM)`. Не убирай эти проверки, иначе вне LMS будет ошибка.

## 3. Тонкости SCORM 1.2, которые уже учтены

1. **`cmi.suspend_data` ограничен ~4096 символами.** Наш JSON крошечный, но если
   будешь хранить больше состояния — следи за объёмом, иначе LMS обрежет строку.

2. **Инициализация API происходит на событие `load`, а не `DOMContentLoaded`.**
   `scorm_api.js` вызывает `LMSInitialize` по `window.load`. Поэтому
   восстановление прогресса (`loadProgress`) тоже повешено на `window.load` —
   чтобы к этому моменту API уже было готово. Если перенести чтение прогресса
   раньше — данные из LMS ещё не будут доступны.

3. **Запись нужно «коммитить».** После `LMSSetValue` LMS не обязана сразу
   сохранять данные. Поэтому `saveProgress` всегда вызывает `SCORM.commit()`
   (это `LMSCommit`). Без коммита прогресс может потеряться при выходе.

4. **Сессия закрывается в `beforeunload`.** `scorm_api.js` вызывает `LMSFinish`
   при закрытии вкладки — иначе некоторые LMS не зачтут попытку.

5. **`lesson_status`.** Пока курс не пройден целиком, пишется `incomplete`;
   когда открыта последняя глава — `completed`. Кнопка «Завершить курс» в конце
   дополнительно вызывает `SCORM.complete()` (ставит `passed`, балл 100).
   Если твоей LMS нужен именно `passed` для зачёта — проверь, доходит ли
   пользователь до кнопки.

## 4. Что важно при доработках

- **Добавляешь новую главу?** Добавь её id в `CHAPTER_ORDER` (в начале `app.js`)
  и добавь карточку `#home-card-N` с бейджем-замком на главной. Логика
  разблокировки и записи подхватится сама.

- **Хочешь сбросить прогресс при тесте?** В консоли браузера:
  `localStorage.removeItem('gostemania_progress')` и перезагрузи страницу.
  Внутри LMS прогресс сбрасывается средствами самой LMS (новая попытка).

- **Меняешь структуру состояния** (`collectState`)? Помни, что у пользователей
  уже может быть сохранён старый формат. `loadProgress` написан «защищённо»
  (проверяет типы и игнорирует мусор), сохраняй этот подход.

- **Тестировать SCORM «по-настоящему»** лучше через SCORM Cloud
  (cloud.scorm.com, есть бесплатный тариф) — загрузил ZIP, прошёл, вышел, зашёл
  снова и проверил, что прогресс восстановился.

## 5. Краткая шпаргалка по API (из `scorm_api.js`)

```js
SCORM.init();                       // LMSInitialize — на window.load
SCORM.set('cmi.suspend_data', str); // LMSSetValue
SCORM.get('cmi.suspend_data');      // LMSGetValue -> string
SCORM.commit();                     // LMSCommit  — обязательно после set
SCORM.finish();                     // LMSFinish  — на beforeunload
SCORM.complete();                   // passed + score 100 + finish (кнопка в конце)
```
